### Post-rename Java-package corruption fix-up

The macOS/BSD `sed` engine doesn't honour `\b` word boundaries, so the
`\.evaluation\b` / `\.solicitation\b` segment rules (intended to fire BEFORE
the bare lowercase rule per `rename-rules.yml` ordering) didn't match, and the
later `s|evaluation|redaction_review|g` + `s|solicitation|foia_request|g`
passes rewrote `.evaluation` → `.redaction_review` (with underscore — invalid
Java identifier) and `.solicitation` → `.foia_request` (same shape) in 78
Java files. A follow-up targeted sed restored the correct no-underscore form
(`.redactionreview`, `.foiarequest`) before commit 1. **Reported as a
DEFERRED gap for SKILL §3 (sed engine portability) — not patched here per
parallel-run isolation rules.**

## bug_sketch field adaptations (FOIA domain)

Per task input, FOIA stakeholders are FOIA Requester, FOIA Officer, Records
Custodian, Agency Reviewer; entities are FoiaRequest, ResponsivePages,
ExemptionDetermination, ReleasePackage. The following pair-unique-debt
`bug_sketch` fields were adapted from the pool YAML's generic placeholders:

- **sec-sql-string-concat-aspect-search:** the pool YAML's example status values
  are aspect-generic. For FOIA, the documented status enum is
  `PROPOSED | UNDER_REVIEW | APPROVED_FOR_RELEASE | WITHHELD` — these are
  redaction-workflow states, not grants-management application states. The
  injected `SearchRepository.searchByStatus(...)` carries this context in the
  Javadoc.
- **rel-retry-without-backoff:** domain framing is the redaction-proposer
  fanning out over multi-thousand-page ResponsivePages uploads — the
  thundering-herd is realistic against Bedrock when an agency uploads a
  large responsive-document set in one shot.
- **obs-error-rethrown-stack-lost:** the chained `ClientError` framed as a
  `ThrottlingException` an SRE needs to see to dial concurrency back during
  multi-thousand-page ResponsivePages runs.
- **ai-bedrock-no-cost-limit:** framed against D-050 (single shared
  Karsun-paid AWS account in W5) — adversarial requester drains the account
  via responsive-document fan-out. HITL #7 (W5 Wed AIOps auto-remediation
  authority) is the cohort's remediation surface.
