## Decision

**This pair commits `foia-processing` (FOIA Request Processing) as their Pair
Project aspect.**

- Primary entity: `FoiaRequest`
- Review entity: `RedactionReview`
- Workflow stages: intake-triage → exemption-analysis → redaction-proposal → hitl-review → response → appeal
- Corpus type: FOIA precedent (5 USC 552 + 28 CFR 16 DOJ FOIA regs)
- Key stakeholders: FOIA Officer, General Counsel, Requester (citizens / journalists / watchdog orgs), OIP (DOJ oversight)
- Agent shape: multi-agent (proposer + reviewer, HITL on irreversible release decisions per `karsun-domain-aspects.yml` ai_adoption_phase_1_seeds)

3. AIOps on response-timeline drift (which FoiaRequests are at statutory-miss risk against the 20-working-day clock?)

## Distinctness from training-project (acquire-gov)

acquire-gov ships solicitation-drafting + evaluation orchestration — the **agency-friendly**
audit posture (Contracting Officer + vendor are both trusted parties working a shared
solicitation). FOIA inverts this: the **requester may be adversarial** (journalists,
watchdog orgs, opposition researchers, foreign-intelligence proxies). The core capability
shifts from drafting to **redaction reliability** — false negatives leak; false positives
trigger appeals. Statutory timeline (20 working days) dominates NFRs in a way no
acquire-gov workflow does. Different stakeholders, different threat model, different
corpus (5 USC 552 case law vs FAR Part 15).
