## Issues observed during reshape

### Java-package corruption from BSD sed `\b` non-portability

The `rename-rules.yml` Phase 1d rules use `\.solicitation\b` /
`\.evaluation\b` with the intent of firing BEFORE the bare lowercase rules.
macOS/BSD `sed` does NOT honour `\b` word boundaries — so these regex rules
matched nothing, and the subsequent `s|evaluation|redaction_review|g` /
`s|solicitation|foia_request|g` passes rewrote `.evaluation` →
`.redaction_review` (with underscore — invalid Java identifier) and
`.solicitation` → `.foia_request` in 78 Java files.

A follow-up targeted `sed` pass restored the correct no-underscore form
(`.redactionreview`, `.foiarequest`) before commit 1. **Not patched in the
SKILL or rename-rules.yml per parallel-run isolation rules** — flagged in
the return summary as a DEFERRED gap for the SKILL maintainer.
